# Task Management API — Hono + Repository Pattern

## Stack

- **Runtime:** Bun (หรือ Node.js 20+)
- **Framework:** Hono v4
- **ORM:** Drizzle ORM + postgres.js
- **Database:** PostgreSQL (Supabase)
- **Auth:** JWT (jose) + bcrypt (pgcrypto)
- **Validation:** Zod + @hono/zod-validator

## Architecture: Repository Pattern

```
HTTP Request
    ↓
requestIdMiddleware  (inject X-Request-ID, log request)
    ↓
rateLimit           (10/min login | 100/min API)
    ↓
Route               (zValidator → parse & validate input)
    ↓
authMiddleware      (verify JWT → inject ctx.user)
    ↓
Service             (business logic, orchestration)
    ↓
Repository          (DB query only — returns typed data or throws AppError)
    ↓
Drizzle ORM → PostgreSQL
    ↓
Route               (ok() / created() / fail())
    ↓
errorMiddleware     (catch AppError / ZodError / unknown → JSON)
    ↓
HTTP Response
```

## Quick Start

```bash
# 1. Install
bun install

# 2. Setup env
cp .env.example .env
# แก้ DATABASE_URL, JWT_SECRET, JWT_REFRESH_SECRET

# 3. Run DB (fresh install)
psql $DATABASE_URL -f schema.sql

# 4. Seed (idempotent — รันกี่ครั้งก็ได้)
bun run db:seed

# 5. Dev server
bun run dev
```

## Adding a new domain

1. **Schema** → `src/db/schema/<domain>.schema.ts` + export ใน `index.ts`
2. **Repository** → `src/repositories/<domain>.repository.ts`
3. **Service** → `src/services/<domain>.service.ts`
4. **Validator** → `src/validators/<domain>.validator.ts`
5. **Route** → `src/routes/<domain>.route.ts`
6. **Register** → `src/app.ts` `.route('/api/<domain>', router)`
